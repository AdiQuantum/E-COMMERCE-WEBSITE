▶ NUMBERED FLOW — HOW A REQUEST TRAVELS THROUGH THE SYSTEM

1
User opens browser and types the app URL. Browser sends an HTTPS GET request to the hosting server (Vercel). Browser → https://yourapp.vercel.app
2
React SPA loads in the browser. React Router reads the URL and renders the matching Page component (e.g., HomePage). Vercel CDN → React bundle → DOM rendered
3
User performs an action (e.g., clicks "Browse Products"). The Page component triggers an API service function. ProductsPage.jsx → productService.js → getProducts()
4
Axios sends HTTP request to the Express backend. JWT token attached in Authorization header if user is logged in. GET /api/products?category=electronics
5
Express server receives request on server.js. It passes through the middleware pipeline before reaching the route. Render.com → server.js → middleware pipeline
6
Middleware runs: CORS check → JSON parser → JWT verification (authMiddleware) → Admin role check (if admin route). authMiddleware.js → req.user = decoded JWT payload
7
Route handler matches the URL and HTTP method. It calls the appropriate controller function. productRoutes.js → GET / → productController.getAllProducts()
8
Controller executes business logic: validates inputs, applies filters, pagination, and calls the Mongoose model. productController.js → Product.find({category}) → .sort() → .limit()
9
Mongoose Model queries MongoDB Atlas. The schema validates data. Query result is returned as JavaScript objects. Product.js (schema) → MongoDB Atlas → documents returned
10
MongoDB Atlas returns data to Mongoose, which maps it to JavaScript objects and sends them back to the controller. Atlas cluster → Mongoose → JS objects → controller
11
Controller sends JSON response back to the frontend with appropriate HTTP status code (200, 201, 400, 401, 403, 404, 500). res.status(200).json({ success: true, data: products })
12
Axios receives JSON. API service passes data to State Management (Redux/Context). State update triggers React re-render. productService.js → dispatch(setProducts) → Redux store
13
React re-renders the UI with the new data. User sees the product list. Spinner is hidden. Toast shown if needed. ProductsPage.jsx reads Redux state → renders ProductCard components
